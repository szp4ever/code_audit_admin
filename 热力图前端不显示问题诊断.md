# 热力图前端不显示问题诊断

## 问题描述
后端日志显示已返回12条数据，但前端没有显示热力图。

## 已添加的调试日志
已在 `apps/web-antd/src/views/dashboard/analytics/user-operation-heatmap.vue` 中添加了详细的调试日志。

**请按以下步骤操作：**

1. **打开浏览器开发者工具（F12）**
2. **切换到 Console（控制台）标签**
3. **刷新页面，查看以 `🔍 [热力图]` 开头的日志**

## 可能原因及解决方案

### 1. 后端返回数据格式不正确（最可能）

**检查方法：**
- 在浏览器 Network 标签中查看请求 `/api/taskmanagement/stats/user-operation-heatmap` 的响应
- 查看控制台日志中的 `🔍 [热力图] 接收到数据:` 输出

**后端应该返回的格式：**
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": [
    {
      "module": "system:user",
      "timeSlot": "2026-02-22",
      "count": 15
    }
  ]
}
```

**常见错误：**
- ❌ 后端直接返回数组：`[{...}, {...}]` （缺少包装）
- ❌ 后端返回的字段名不匹配：`moduleName` vs `module`
- ❌ 后端返回的 `count` 是字符串而不是数字

**解决方案：**
确保后端 Controller 使用统一响应格式：
```java
@GetMapping("/stats/user-operation-heatmap")
public R<List<UserOperationHeatmapItem>> getUserOperationHeatmap(...) {
    List<UserOperationHeatmapItem> result = ...;
    return R.ok(result);  // ✅ 正确：会自动包装为 {code: 200, data: [...]}
}
```

**错误示例：**
```java
// ❌ 错误：直接返回列表，没有包装
return result;

// ❌ 错误：手动包装但格式不对
return new R<>(200, "成功", result);
```

---

### 2. 数据字段名不匹配

**检查方法：**
查看控制台日志中的 `🔍 [热力图] 数据字段检查:` 输出

**前端期望的字段：**
- `module` (string) - 功能模块标识
- `timeSlot` (string) - 时间段，格式：`YYYY-MM-DD`
- `count` (number) - 使用频次，必须是数字类型

**如果字段名不匹配，后端需要调整：**
```java
// 确保 DTO 类的字段名与前端一致
public class UserOperationHeatmapItem {
    private String module;      // ✅ 必须是 module
    private String timeSlot;    // ✅ 必须是 timeSlot
    private Integer count;      // ✅ 必须是 count，且是数字类型
}
```

---

### 3. 数据类型错误

**检查方法：**
查看控制台日志中的 `countType` 输出

**问题：**
- `count` 字段是字符串 `"15"` 而不是数字 `15`

**解决方案：**
确保后端返回的 `count` 是数字类型：
```java
item.setCount(count.intValue());  // ✅ 确保是 Integer 类型
```

---

### 4. 前端处理数据时出错

**检查方法：**
查看控制台是否有 `❌ [热力图]` 开头的错误日志

**可能的问题：**
- 数据为空数组但前端没有正确处理
- 数据格式正确但 ECharts 渲染失败

**解决方案：**
查看完整的错误堆栈信息，根据具体错误进行修复。
